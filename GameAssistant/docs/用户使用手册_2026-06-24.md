## 一、系统概述

GameAssistant 是一个游戏辅助标注与训练平台，提供从图片/视频标注、数据集管理、模型训练到推理部署的完整 AI 模型开发流程。

### 1.1 主要功能模块

| 模块 | 说明 |
|------|------|
| 图片管理 | 上传图片、批量导入、视频抽帧 |
| 标注管理 | 标注项目管理、图片标注、类别管理、审核流程 |
| 模板管理 | 模板图像上传、模板匹配参数配置、匹配测试 |
| 数据集管理 | 数据集与版本管理、训练/验证/测试集划分、数据增强 |
| 模型管理 | 模型上传、版本切换、模型详情与评估曲线 |
| 训练管理 | 发起 YOLO 训练任务、实时日志查看、指标图表 |
| 推理测试 | 设备实时推理、图片/视频推理、推理历史记录 |
| 用户管理 | 用户账号管理（管理员专属） |

### 1.2 YOLO 全链路训练流程

本平台支持完整的 YOLO 模型训练链路，从数据准备到模型部署的完整流程如下：

│                                          • 生成YOLO格式         │

## 三、快速开始：自定义场景 YOLO 模型训练

### 3.1 整体操作流程

使用本平台训练自定义 YOLO 模型的标准流程如下：

```
步骤 1：准备图片数据
    ├── 上传单张图片
    ├── 批量上传（ZIP）
    └── 视频抽帧

步骤 2：创建标注类别
    ├── 定义类别名称
    └── 设置 YOLO ID（从0开始递增）

步骤 3：标注图片
    ├── 绘制边界框
    └── 选择类别

步骤 4：创建数据集
    ├── 创建数据集
    ├── 创建版本
    └── 添加图片并划分 train/val/test

步骤 5：生成 YOLO 数据集
    └── 系统自动生成 ZIP 包（含 images/labels/data.yaml）

步骤 6：发起训练
    ├── 选择数据集版本
    ├── 配置训练参数
    └── 启动训练

步骤 7：监控训练
    ├── 查看实时日志
    ├── 监控指标曲线
    └── 停止/恢复训练

步骤 8：使用模型
    ├── 查看训练结果
    ├── 激活模型
    └── 推理测试
```

### 3.2 详细操作指南

#### 步骤 1：准备图片数据

**方式 A：上传图片**

1. 进入"图片管理"页面
2. 点击"上传图片"按钮
3. 选择本地图片文件（支持 JPG、PNG、WebP）
4. 等待上传完成

**方式 B：批量上传**

1. 进入"图片管理"页面
2. 点击"批量上传"按钮
3. 选择包含多张图片的 ZIP 文件
4. 系统自动解压并上传所有图片

**方式 C：视频抽帧**

1. 进入"图片管理"页面
2. 找到"视频提取"面板
3. 上传视频文件（支持 MP4、AVI、MKV 等格式）
4. 配置提取策略：
   - **按时间间隔**：每 N 秒提取一帧（如每 1 秒）
   - **按数量**：共提取 N 张图片（如提取 100 张）
   - **场景切换**：检测画面大幅变化时提取
5. 点击"开始提取"并等待完成

> **建议**：对于游戏场景，推荐使用"场景切换"策略，可以自动提取包含不同游戏状态的帧。

#### 步骤 2：创建标注类别

1. 进入"标注管理" → "类别管理"页面
2. 点击"新建类别"按钮
3. 填写类别信息：
   - **类别名称**：如 "player"、"enemy"、"coin"、"boss" 等
   - **显示颜色**：选择一个便于区分的颜色
   - **快捷键**：设置一个快捷键方便标注时快速切换
   - **YOLO ID**：从 0 开始，每个类别必须唯一
4. 点击"保存"

**YOLO ID 规范**：
- 必须从 0 开始
- 每个类别 ID 必须唯一
- 不能跳过数字（如 0, 1, 2, 不能跳过 1 直接用 2）
- ID 顺序应与类别重要性或出现频率相关

**示例类别配置**：

| 类别名称 | YOLO ID | 快捷键 | 颜色 |
|----------|---------|--------|------|
| player | 0 | p | 蓝色 |
| enemy | 1 | e | 红色 |
| coin | 2 | c | 黄色 |
| obstacle | 3 | o | 紫色 |

#### 步骤 3：标注图片

1. 进入"标注管理" → "图片标注"页面
2. 选择标注项目（如果没有，先创建项目并添加图片）
3. 切换到"画框模式"
4. 从左侧列表选择要标注的图片
5. 在图片上按住鼠标左键拖动，绘制边界框
6. 松开鼠标后，从弹出面板选择对应的类别
7. 重复以上步骤，标注图片中的所有目标

**标注技巧**：
- 边界框应紧贴目标边缘，不要留太多空白
- 如果两个目标重叠，只标注外面的那个
- 一张图片可以标注多个同类目标
- 使用快捷键可以快速切换类别

**编辑和删除标注**：
- 切换到"编辑模式"
- 点击选中已有的标注框
- 拖动边框调整大小
- 按 Delete 键删除选中的标注

#### 步骤 4：创建数据集

1. 进入"数据集管理"页面
2. 点击"创建数据集"按钮
3. 填写数据集名称（如 "我的游戏数据集"）和描述
4. 点击"确定"

**创建数据集版本**：

1. 在左侧列表选择刚才创建的数据集
2. 点击"创建版本"按钮
3. 填写版本名称（如 "v1.0"）
4. 配置划分比例：
   - **训练集**：默认 80%
   - **验证集**：默认 10%
   - **测试集**：默认 10%
5. 点击"确定"

**添加图片到版本**：

1. 在版本管理中点击"管理图片"按钮
2. 在左侧图片库中勾选要添加的图片
3. 在右侧选择划分类型（train/val/test）
4. 点击"添加选中图片"

> **注意**：添加图片后，系统会自动从标注中提取涉及的类别 ID。

#### 步骤 5：生成 YOLO 数据集

1. 在数据集版本管理中，找到目标版本
2. 点击"生成 YOLO"按钮
3. 等待系统处理完成（生成 ZIP 文件）
4. 可以点击"下载"按钮下载生成的 YOLO 数据集

**生成的内容结构**：

```
yolo_dataset.zip
├── train/
│   ├── images/
│   │   ├── image1.jpg
│   │   ├── image2.jpg
│   │   └── ...
│   └── labels/
│       ├── image1.txt
│       ├── image2.txt
│       └── ...
├── val/
│   ├── images/
│   └── labels/
├── test/
│   ├── images/
│   └── labels/
└── data.yaml
```

**data.yaml 文件示例**：

```yaml
path: .
train: train/images
val: val/images
test: test/images
names:
  - player
  - enemy
  - coin
  - obstacle
```

#### 步骤 6：发起训练

1. 进入"训练管理"页面
2. 点击"发起训练"按钮
3. 配置训练参数：
   - **选择数据集版本**：选择刚才准备好的数据集版本
   - **任务名称**：为训练任务起一个名字
   - **基础架构**：选择 YOLO 模型大小
     - `yolov8n` - nano（最快，精度最低）
     - `yolov8s` - small
     - `yolov8m` - medium（推荐）
     - `yolov8l` - large
     - `yolov8x` - xlarge（最慢，精度最高）
   - **训练轮次 (Epochs)**：50-300，默认 100
   - **批次大小 (Batch Size)**：根据显存调整，6GB 显存建议 8-16
   - **图像尺寸**：640 或 1280，默认 640
   - **初始学习率 (lr0)**：默认 0.01
   - **最终学习率 (lrf)**：默认 0.01
   - **早停耐心值**：连续 N 轮无提升则停止，默认 50
   - **数据增强**：
     - Mosaic：默认开启
     - MixUp：默认关闭
     - 水平翻转：默认开启
4. 点击"开始训练"

**参数调优建议**：

| 场景 | epochs | batch_size | img_size | 说明 |
|------|--------|------------|----------|------|
| 快速测试 | 10-20 | 16 | 640 | 验证流程是否正常 |
| 标准训练 | 100 | 16 | 640 | 平衡速度和精度 |
| 高精度 | 300 | 8 | 1280 | 需要更多显存 |
| 小目标 | 200 | 8 | 1280 | 使用更大分辨率 |

#### 步骤 7：监控训练

训练开始后，可以在"训练管理"页面实时监控：

**查看训练进度**：
- 左侧列表显示所有训练任务
- 点击任务可查看详情
- 进度条显示当前 epoch / 总 epoch

**查看实时指标**：
- **mAP50**：验证集上的平均精度（IoU=0.5），越高越好
- **mAP50-95**：验证集上的平均精度（IoU=0.5~0.95），综合指标
- **precision**：精确率
- **recall**：召回率
- **Box Loss**：边界框回归损失，越低越好
- **Cls Loss**：分类损失，越低越好

**查看训练日志**：
1. 点击训练任务的"查看"按钮
2. 切换到"训练日志" Tab
3. 查看每个 epoch 的详细输出

**停止训练**：
- 如果训练时间过长或效果不理想，可以点击"停止"按钮
- 训练停止后，可以使用当前最好的模型

#### 步骤 8：使用训练好的模型

**查看训练结果**：
1. 进入"模型管理"页面
2. 找到训练完成时自动创建的模型
3. 点击"详情"查看评估指标

**评估指标解读**：

| 指标 | 说明 | 合格标准 |
|------|------|----------|
| mAP50 | IoU=0.5 时的平均精度 | > 0.5 为可用 |
| mAP50-95 | IoU 0.5-0.95 的平均精度 | > 0.3 为可用 |
| precision | 预测正确的比例 | > 0.7 为良好 |
| recall | 找出所有目标的比例 | > 0.6 为良好 |

**激活模型**：
1. 找到要使用的模型
2. 点击"设为默认"按钮
3. 该模型将用于后续的推理测试

**推理测试**：
- 进入"推理测试"页面
- 选择推理方式：
  - **设备推理**：连接 Android 设备实时检测
  - **图片推理**：上传图片进行检测
  - **视频推理**：上传视频逐帧检测

## 四、启动应用

### 4.1 方式一：Docker 一键启动（推荐）

适用场景：初次部署、迁移环境、快速体验。

**前提条件**：已安装 Docker 和 Docker Compose。

**操作步骤**：

1. 打开终端，进入项目目录：

```bash
cd c:\project\modelTraining\GameAssistant\web
```

2. 复制环境变量配置文件：

```bash
cp .env.example .env
```

3. 启动所有服务（MySQL 数据库 + 后端 + 前端）：

```bash
docker-compose up -d
```

4. 等待服务启动（约 10-30 秒），验证服务状态：

```bash
docker-compose ps
```

预期输出应包含三个运行中的服务：

| 服务名 | 端口 | 状态 |
|--------|------|------|
| mysql | 3307 | Up |
| backend | 8000 | Up |
| frontend | 3000 | Up |

5. 停止所有服务：

```bash
docker-compose down
```

---

### 4.2 方式二：手动分步启动

适用场景：本地开发、需要修改代码、单步调试。

#### 步骤 1：启动 MySQL 数据库

确保本地已安装 MySQL 8.0，并创建数据库和用户：

```sql
-- 登录 MySQL
mysql -u root -p 

-- 创建数据库和用户
CREATE DATABASE IF NOT EXISTS gameassistant CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER IF NOT EXISTS 'gameassistant'@'%' IDENTIFIED BY 'gameassistant';
GRANT ALL PRIVILEGES ON gameassistant.* TO 'gameassistant'@'%';
FLUSH PRIVILEGES;
```

#### 步骤 2：启动后端服务

```bash
cd c:\project\modelTraining\GameAssistant\web\backend

# 安装 Python 依赖
python -m pip install -r requirements.txt

# 安装 ultralytics（YOLO 训练必需）
pip install ultralytics

# 启动后端（开发模式，自动重载）
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

**预期结果**：终端输出类似以下内容：

```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Application startup complete.
INFO:     Database initialized successfully
INFO:     Admin user created/verified
```

> **注意**：首次启动时，后端会自动执行数据库初始化（建表）和创建默认管理员账号，无需手动执行 SQL 脚本。

#### 步骤 3：启动前端服务

另开一个终端窗口：

```bash
cd c:\project\modelTraining\GameAssistant\web\frontend

# 安装 Node.js 依赖（如尚未安装）
npm install

# 启动开发服务器
npm run dev
```

#### 步骤 4：访问应用

打开浏览器，访问：

```
http://localhost:3000
```

## 六、页面操作指南

### 6.1 图片管理

**入口**：顶部导航栏 → "图片管理"

**功能说明**：管理游戏截图和素材图片，支持单张上传、批量 ZIP 导入，以及从视频中提取关键帧。

#### 操作 1：上传单张图片

1. 点击"上传图片"按钮
2. 在文件选择器中选择一张图片文件（支持 JPG、PNG、WebP）
3. 等待上传完成，图片自动出现在下方网格中

**理想结果**：图片以卡片形式显示在网格中，包含缩略图、文件名、文件大小、来源标签。

#### 操作 2：批量上传（ZIP）

1. 点击"批量上传"按钮
2. 选择一个包含多张图片的 ZIP 压缩包
3. 系统自动解压并上传所有图片

**理想结果**：上传完成后网格中显示所有图片，总数与 ZIP 内文件数一致。

#### 操作 3：视频抽帧

1. 在页面顶部找到"视频提取"面板
2. 点击"上传视频"，选择 MP4/AVI 等格式的视频文件
3. 配置提取策略：
   - **按时间间隔**：每 N 秒提取一帧
   - **按数量**：共提取 N 张
   - **场景切换**：检测画面大幅变化时提取
4. 点击"开始提取"
5. 等待提取任务完成（可刷新查看进度）

**理想结果**：提取完成后图片列表中新增来自该视频的所有帧图像，来源标签显示为"视频提取"。

#### 操作 4：筛选图片

1. 在页面顶部来源下拉框中选择筛选条件
2. 选项包括：全部 / 手动上传 / 视频提取
3. 点击"刷新"按钮重新加载列表

**理想结果**：列表中仅显示符合筛选条件的图片。

#### 操作 5：删除图片

1. 点击图片卡片上的复选框选中图片（可多选）
2. 点击"删除选中"按钮
3. 在确认对话框中点击"确定"

**理想结果**：选中的图片从列表中消失，图片计数相应减少。

#### 操作 6：查看大图

1. 点击任意图片卡片
2. 弹出全屏图片查看器（灯箱）
3. 使用键盘左右方向键翻页，ESC 键关闭

### 6.2 标注管理

**入口**：顶部导航栏 → "标注管理"

标注管理包含四个子页面，通过顶部 Tab 切换。

#### 6.2.1 标注项目

**功能说明**：管理标注任务项目，分配给不同用户进行标注和审核。

##### 创建项目

## 八、常见问题

### Q1：启动后端时报数据库连接错误

```
sqlalchemy.exc.OperationalError: (pymysql.err.OperationalError) 
(2003, "Can't connect to MySQL server")
```

**原因**：MySQL 服务未启动或连接信息错误。

**解决**：

1. 确认 MySQL 服务正在运行
2. 检查 `config.py` 中的 `DB_URL` 是否与实际数据库配置一致
3. 如果使用 Docker，确认 docker-compose 中的 MySQL 容器已启动

### Q2：登录时提示用户名或密码错误

**原因**：默认账号密码不正确，或数据库未初始化。

**解决**：

1. 确认使用正确的默认账号：`admin` / `admin123`
2. 检查后端日志是否输出了 `Admin user created` 信息
3. 如账号不存在，删除数据库重建或手动调用初始化接口

### Q3：前端访问 `http://localhost:3000` 显示空白

**原因**：前端开发服务器未启动或端口被占用。

**解决**：

1. 确认前端终端无报错
2. 换一个端口启动：`npm run dev -- --port 3001`
3. 检查是否有其他程序占用了 3000 端口

### Q4：训练任务创建后一直处于"排队中"

**原因**：GPU 资源被占用，或 CUDA 环境未配置。

**解决**：

1. 检查机器是否有 NVIDIA GPU：`nvidia-smi`
2. 确认已安装 CUDA 和 cuDNN
3. 确认已安装 ultralytics：`pip install ultralytics`
4. 检查训练日志是否有报错信息

### Q5：训练时报错 "ultralytics not installed"

**原因**：未安装 ultralytics 库。

**解决**：

```bash
pip install ultralytics
```

### Q6：训练时报错 "CUDA out of memory"

**原因**：显存不足，batch size 设置过大。

**解决**：

1. 减小 batch size（从 16 降到 8 或 4）
2. 减小图像尺寸（从 1280 降到 640）
3. 使用更小的模型（如 yolov8n 代替 yolov8x）

### Q7：生成 YOLO 数据集后训练失败

**原因**：数据集版本状态不是 "ready"。

**解决**：

1. 确认已为版本添加图片
2. 确认图片已有标注
3. 点击"生成 YOLO"按钮重新生成

### Q8：模板匹配测试结果为空

**原因**：模板图像与测试图片不匹配，或置信度阈值过高。

**解决**：

1. 降低置信度阈值（如从 0.8 降到 0.5）
2. 确保模板图像清晰、背景简洁
3. 尝试不同的匹配方法（TM_CCOEFF_NORMED 通常效果最好）

### Q9：设备推理显示"未检测到设备"

**原因**：USB 调试未开启或 ADB 驱动未安装。

**解决**：

1. 在 Android 设备上：设置 → 关于手机 → 点击"版本号"7次开启开发者模式
2. 设置 → 开发者选项 → 开启 USB 调试
3. 在电脑上安装对应机型的 USB 驱动
4. 在终端执行 `adb devices` 验证连接

### Q10：标注框绘制不准怎么办

**解决**：

1. 使用编辑模式调整边界框位置和大小
2. 确保边界框紧贴目标边缘，不要留太多空白
3. 删除不准确的标注重新绘制
