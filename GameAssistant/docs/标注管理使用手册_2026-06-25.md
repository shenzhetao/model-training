> 文档版本：v1.2
> 生成日期：2026-06-25
> 适用版本：GameAssistant Web

---

## 一、模块概述

标注管理模块是平台的核心功能，用于管理 YOLO 目标检测模型的训练数据准备工作。模块包含 **4 个标签页**，通过顶部 Tab 切换：

| 标签页 | 功能说明 |
|--------|----------|
| 标注项目 | 项目的创建、状态管理、进度追踪 |
| 图片标注 | 在图片上绘制边界框并标注类别 |
| 类别管理 | 定义检测目标的类别（如：攻击按钮、金币） |
| 审核管理 | 审核标注结果，通过或退回 |

---

## 二、标签页之间的关系

```
┌─────────────────────────────────────────────────────────────────┐
│                        标注管理                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌──────────────┐    ┌──────────────┐                         │
│  │  类别管理    │───→│  标注项目    │                         │
│  │  (定义类别)  │    │  (创建项目)  │                         │
│  └──────────────┘    └──────┬───────┘                         │
│                             │                                  │
│                             ↓                                  │
│                       ┌──────────────┐                         │
│                       │  图片标注    │ ← 添加图片到项目         │
│                       │  (标注工作)  │ ← 绘制边界框             │
│                       └──────┬───────┘                         │
│                             │                                  │
│                             ↓                                  │
│                       ┌──────────────┐                         │
│                       │  审核管理    │                         │
│                       │  (审核结果)  │                         │
│                       └──────────────┘                         │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 数据流向

1. **类别管理** → 定义要检测的目标类型
2. **标注项目** → 创建任务并关联类别
3. **图片标注** → 向项目中添加图片并绘制标注框
4. **审核管理** → 检查标注质量，通过或退回

## 三、各标签页详细说明

### 3.1 类别管理

**用途**：定义检测目标的所有类别，每个类别对应 YOLO 模型中的一个 class_id。

#### 创建类别

1. 点击「**新建类别**」按钮
2. 填写信息：
   - **类别名称**：英文标识，如 `btn_attack`
   - **显示名称**：中文展示，如「攻击按钮」
   - **描述**（可选）：类别的详细说明
   - **颜色**：选择标注框的显示颜色
   - **YOLO 类别 ID**：从 0 开始的数字，如 `0`、`1`、`2`
   - **快捷键**（可选）：设置一个字母键，标注时快速切换

#### 类别 ID 注意事项

> **重要**：YOLO 类别 ID 必须从 0 开始，且不能重复。如果有 3 个类别，则 ID 分别为 `0`、`1`、`2`。

---

### 3.2 标注项目

**用途**：创建和管理标注任务项目，追踪标注进度。

#### 创建项目

1. 点击「**新建项目**」按钮
2. 填写项目信息：
   - **项目名称**：给项目起个名字
   - **描述**（可选）：项目的详细说明
   - **包含类别**：选择该项目需要标注的类别（可多选）
3. 点击「**确定**」创建

#### 修改项目状态

1. 在项目列表中点击「**编辑**」按钮
2. 在弹窗中找到「**状态**」下拉框
3. 选择目标状态：
   - **草稿**：项目刚创建，尚未开始
   - **进行中**：标注工作正在进行
   - **已完成**：标注工作完成，等待审核
4. 点击「**确定**」保存

#### 项目状态说明

| 状态 | 含义 |
|------|------|
| 草稿 | 项目刚创建，未开始标注 |
| 进行中 | 正在进行标注工作 |
| 已完成 | 所有图片标注完成，等待提交审核 |
| 待审核 | 已提交审核，等待审核人员检查 |
| 已退回 | 审核不通过，需要重新标注 |
| 已审核 | 审核通过，标注质量合格 |

#### 项目操作按钮

| 按钮 | 功能 |
|------|------|
| 标注 | 进入「图片标注」页面，开始标注工作 |
| 编辑 | 修改项目信息或状态 |
| 删除 | 删除项目（需确认） |

---

### 3.3 图片标注（核心工作区）

**用途**：在图片上绘制边界框并标注类别。

#### 工作流程

```
┌─────────────────────────────────────────────────────────┐
│                    图片标注工作流程                      │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  1. 从左侧图片列表选择一张图片                           │
│           ↓                                              │
│  2. 点击「添加到项目」按钮                               │
│           ↓                                              │
│  3. 选择标注类别                                         │
│           ↓                                              │
│  4. 点击「画框」模式                                      │
│           ↓                                              │
│  5. 在图片上拖动鼠标绘制边界框                           │
│           ↓                                              │
│  6. 点击「保存」按钮                                     │
│           ↓                                              │
│  7. 重复步骤 1-6 标注更多图片                           │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

#### 详细操作步骤

**第一步：选择图片并添加到项目**

1. 从左侧「图片列表」中点击选择一张图片
2. 点击「**添加到项目**」按钮
3. 系统提示「图片已添加到项目」

> **注意**：必须先将图片添加到项目，标注后进度才会增加。如果图片不在项目中，标注框会保存但不会更新项目进度。

**第二步：标注图片**

1. 从顶部类别面板选择一个类别（点击颜色标签）
2. 点击「**画框**」按钮切换到画框模式
3. 在图片上按住鼠标左键拖动，绘制矩形框
4. 松开鼠标后，标注框被保存
5. 点击「**保存**」按钮确认

#### 画布工具说明

| 按钮 | 功能 |
|------|------|
| 浏览 | 查看模式，不可绘制 |
| 画框 | 绘制矩形边界框 |
| 编辑 | 拖动调整已绘制的框位置 |

#### 标注完成后

1. 回到「标注项目」标签页
2. 查看该项目的「**进度**」列，数值已更新
3. 当所有图片标注完成后，点击「**提交审核**」按钮

#### 提交审核

1. 确保已选择当前项目
2. 点击「**提交审核**」按钮
3. 项目状态变为「**待审核**」

---

### 3.4 审核管理

**用途**：审核人员检查标注质量，决定通过或退回。

#### 审核流程

```
┌─────────────────────────────────────────────────────────┐
│                    审核工作流程                          │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  1. 标注员完成标注 → 点击「提交审核」                     │
│           ↓                                              │
│  2. 审核员进入「审核管理」标签页                         │
│           ↓                                              │
│  3. 从审核队列选择项目                                   │
│           ↓                                              │
│  4. 检查标注质量                                         │
│           ↓                                              │
│  5. 点击「通过审核」或「退回修改」                       │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

#### 审核操作

| 操作 | 说明 |
|------|------|
| **通过审核** | 标注质量合格，项目标记为已完成 |
| **退回修改** | 标注有问题，填写退回原因，项目打回重新标注 |

#### 查看退回原因

如果项目被退回，可以在项目详情中查看审核人员填写的反馈信息，根据反馈修改后重新提交。

## 四、完整使用示例

### 场景：标注游戏界面中的按钮

**目标**：训练一个识别游戏中攻击、防御、跳跃按钮的模型

#### Step 1：创建类别

1. 进入「**类别管理**」
2. 创建三个类别：
   - 攻击按钮：YOLO ID = 0
   - 防御按钮：YOLO ID = 1
   - 跳跃按钮：YOLO ID = 2

#### Step 2：创建项目

1. 进入「**标注项目**」
2. 点击「**新建项目**」
3. 填写：
   - 项目名称：游戏按钮标注
   - 包含类别：攻击按钮、防御按钮、跳跃按钮
4. 点击确定

#### Step 3：标注图片

1. 点击项目的「**标注**」按钮
2. 进入「图片标注」页面
3. 选择一张游戏截图
4. 点击「**添加到项目**」
5. 选择类别「攻击按钮」
6. 点击「**画框**」
7. 在攻击按钮位置拖动绘制矩形框
8. 点击「**保存**」
9. 重复 3-8 标注防御按钮和跳跃按钮
10. 切换下一张图片继续标注

#### Step 4：提交审核

1. 所有图片标注完成后
2. 点击「**提交审核**」
3. 状态变为「待审核」

#### Step 5：审核

1. 审核员进入「**审核管理**」
2. 选择待审核项目
3. 检查标注是否准确
4. 点击「**通过审核**」或「**退回修改**」

## 五、常见问题

### Q1：为什么标注保存后项目进度没有增加？

**原因**：图片尚未添加到项目中。

**解决方法**：
1. 选择要标注的图片
2. 点击「**添加到项目**」按钮
3. 然后再进行标注

### Q2：如何修改项目状态？

**解决方法**：
1. 在「标注项目」标签页点击「**编辑**」
2. 在弹窗中找到「状态」下拉框
3. 选择目标状态并保存

### Q3：如何发起审核？

**解决方法**：
1. 完成所有图片标注
2. 在「图片标注」页面点击「**提交审核**」按钮

### Q4：审核被退回后怎么办？

**解决方法**：
1. 查看退回原因（显示在项目详情中）
2. 根据反馈修改标注
3. 重新点击「提交审核」

## 六、数据模型说明

标注管理涉及以下核心数据表：

| 表名 | 说明 |
|------|------|
| `annotation_classes` | 标注类别定义 |
| `annotation_projects` | 标注项目 |
| `annotation_project_images` | 项目与图片的关联关系 |
| `annotations` | 单个边界框标注 |
| `annotation_reviews` | 审核记录 |

**关键字段**：
- `total_images`：项目总图片数
- `annotated_images`：已标注的图片数
- `progress = annotated_images / total_images`
