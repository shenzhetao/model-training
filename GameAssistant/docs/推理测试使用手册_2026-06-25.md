## 一、模块概述

推理测试模块是游戏辅助系统的核心功能，用于在实际场景中测试目标检测模型的效果。模块支持多种推理模式，包括 YOLO 深度学习检测、模板匹配以及两者的混合模式。

模块包含以下主要功能区域：

| 区域 | 功能说明 |
|------|----------|
| 设备推理 | 连接 Android 设备进行实时检测 |
| 视频推理 | 对视频文件进行批量检测 |
| 图片推理 | 对本地图片进行批量检测 |
| 推理历史 | 查看和管理历史检测记录 |
| 检测参数 | 配置检测的置信度和刷新间隔 |

---

## 二、核心概念

### 2.1 推理模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 混合（hybrid） | YOLO + 模板匹配综合使用 | 需要高准确性和召回率 |
| YOLO | 仅使用深度学习模型检测 | 有训练好的模型 |
| 模板（template） | 仅使用模板匹配 | 无 YOLO 模型或特定元素 |

### 2.2 检测参数

| 参数 | 说明 |
|------|------|
| 置信度阈值 | YOLO 检测的最小置信度要求 |
| 自动刷新间隔 | 设备推理自动截图的间隔时间 |
| 多匹配 | 是否允许同一个模板匹配多个结果 |

## 四、各功能详细说明

### 4.1 设备推理（核心功能）

**用途**：连接 Android 设备进行实时的目标检测。

#### 前置准备

**硬件要求**：
- Android 手机或模拟器
- USB 数据线（用于 ADB 连接）

**软件要求**：
- 手机开启开发者选项
- USB 调试模式已启用
- 已安装 ADB 驱动

#### 连接设备

1. 使用 USB 线将 Android 设备连接到电脑
2. 在手机上确认「允许 USB 调试」弹窗
3. 点击页面右上角的「**检测设备**」按钮
4. 系统自动检测并显示设备信息

#### 设备信息显示

连接成功后，页面显示设备详情：

| 字段 | 说明 |
|------|------|
| 设备ID | 设备的唯一标识符 |
| 型号 | Android 设备型号 |
| 分辨率 | 屏幕分辨率（宽 x 高） |
| 状态 | 连接状态 |

#### 截图操作

| 操作 | 说明 |
|------|------|
| 截图 | 获取当前设备屏幕画面 |
| 自动刷新 | 按设定间隔自动截图并检测 |

#### 推理检测

1. 点击「**截图**」或「**截图 + 推理**」按钮
2. 系统获取屏幕画面并进行目标检测
3. 检测结果显示在右侧面板

#### 检测结果解读

| 字段 | 说明 |
|------|------|
| 类别 | 检测到的目标类别名称 |
| 坐标 | 边界框位置 (x, y, 宽, 高) |
| 置信度 | 检测的置信度百分比 |
| 来源 | 检测来源（YOLO 或模板匹配） |

#### 置信度颜色编码

| 置信度范围 | 颜色 | 含义 |
|------------|------|------|
| ≥80% | 绿色 | 高置信度，结果可靠 |
| 60%-80% | 蓝色 | 中等置信度 |
| 40%-60% | 黄色 | 较低置信度 |
| <40% | 红色 | 低置信度，结果可能不准确 |

#### 自动刷新模式

1. 设置自动刷新间隔（2-30 秒）
2. 点击「**自动刷新**」按钮开启
3. 系统按间隔自动截图并进行检测
4. 点击「**停止**」按钮结束自动刷新

#### 导出检测结果

当检测结果存在时，可点击「**导出检测结果**」按钮：
- 导出格式：JSON 文件
- 包含内容：所有检测框的类别、坐标、置信度

### 4.2 视频推理

**用途**：对视频文件进行批量检测，统计每帧的检测结果。

#### 操作步骤

1. 切换到「**视频推理**」标签页
2. 点击「**选择视频文件**」按钮
3. 从本地选择视频文件（支持 mp4、avi、mov 等格式）
4. 系统自动显示视频预览
5. 点击「**开始推理**」按钮

#### 推理结果

| 字段 | 说明 |
|------|------|
| 帧 | 视频帧序号 |
| 检测数 | 该帧检测到的目标数量 |
| 耗时 | 该帧推理耗时（毫秒） |

#### 结果统计

推理完成后，列表显示所有帧的检测统计：
- 每帧检测到的目标数量
- 每帧的推理耗时
- 总检测数量

---

### 4.3 图片推理

**用途**：对本地图片进行批量检测。

#### 操作步骤

1. 切换到「**图片推理**」标签页
2. 点击「**选择图片**」按钮
3. 从本地选择图片文件（最多 10 张，支持 jpg、png 等格式）
4. 从下拉框选择要使用的检测模型
5. 点击「**批量推理**」按钮

#### 推理结果

| 字段 | 说明 |
|------|------|
| 文件名 | 图片名称 |
| 检测数 | 检测到的目标数量 |
| 耗时 | 推理耗时（毫秒） |

#### 结果预览

- 点击列表中的结果项
- 右侧预览区域显示检测后的图片
- 可查看每张图片的具体检测结果

---

### 4.4 推理历史

**用途**：查看和管理历史检测记录。

#### 历史记录内容

| 字段 | 说明 |
|------|------|
| 名称 | 检测对象名称（设备ID/文件名） |
| 来源 | 检测来源（设备/图片/视频） |
| 检测数 | 检测到的目标总数 |
| 耗时 | 推理耗时 |
| 时间 | 检测时间 |
| 操作 | 可执行的操作 |

#### 来源筛选

点击筛选下拉框可按来源筛选：
- 全部
- 设备
- 图片
- 视频

#### 查看详情

1. 点击历史记录的「**查看**」按钮
2. 右侧详情面板显示：
   - 来源类型
   - 推理模式
   - 置信度阈值
   - 检测数量
   - 推理耗时
   - 检测时间
3. 下方的表格显示具体检测结果

#### 删除记录

点击历史记录的「**删除**」按钮：
- 删除单条历史记录
- 释放存储空间

---

### 4.5 检测参数配置

#### 推理模式选择

| 模式 | 说明 |
|------|------|
| 混合 | 综合使用 YOLO 和模板匹配 |
| 模板 | 仅使用模板匹配 |
| YOLO | 仅使用 YOLO 模型 |

#### YOLO 置信度

- 范围：5% - 95%
- 调整方式：拖动滑块
- 默认值：25%
- 说明：置信度越高，结果越准确但召回率越低

#### 自动刷新间隔

- 范围：2 - 30 秒
- 说明：设置自动检测的时间间隔

## 五、完整使用示例

### 场景：使用手机测试游戏界面检测

**目标**：在 Android 设备上实时检测游戏界面中的按钮

#### Step 1：连接设备

1. 准备 Android 手机，打开游戏
2. 使用 USB 线连接电脑
3. 在手机上允许 USB 调试
4. 点击「**检测设备**」按钮
5. 确认设备信息显示正常

#### Step 2：截图并检测

1. 点击「**截图 + 推理**」按钮
2. 等待检测完成
3. 查看检测结果：
   - 统计检测到的元素数量
   - 查看每个元素的类别和置信度
   - 确认检测是否准确

#### Step 3：调整参数

1. 如果检测数量过多，尝试提高置信度
2. 如果漏检目标，尝试降低置信度
3. 根据需要切换推理模式

#### Step 4：使用自动刷新

1. 设置自动刷新间隔（如 5 秒）
2. 点击「**自动刷新**」按钮
3. 系统自动进行检测
4. 观察实时检测效果
5. 点击「**停止**」结束

#### Step 5：导出结果

1. 检测完成后点击「**导出检测结果**」
2. 保存 JSON 文件到本地
3. 可用于后续分析或脚本集成

### 场景：对视频进行批量检测

**目标**：检测游戏视频中的所有按钮点击

3. 分析检测耗时分布

## 六、常见问题

### Q1：设备连接失败怎么办？

**排查步骤**：
1. 检查 USB 线是否正常（可尝试更换数据线）
2. 确认手机上已允许 USB 调试
3. 检查 ADB 驱动是否安装
4. 尝试重新插拔 USB 线
5. 在终端执行 `adb devices` 检查设备是否识别

### Q2：如何提高检测准确性？

**建议**：
1. YOLO 模式：确保模型已针对目标场景训练
2. 模板模式：确保模板图片清晰且与目标一致
3. 调整置信度阈值
4. 混合模式：综合两种方法的结果

### Q3：检测速度很慢怎么办？

**优化建议**：
1. 减小截图分辨率
2. 使用更轻量的模型
3. 减少同时使用的模板数量
4. 关闭不需要的检测模式

### Q4：如何选择合适的置信度？

**参考**：
| 场景 | 建议置信度 | 说明 |
|------|------------|------|
| 高精度需求 | 50%-70% | 减少误检 |
| 平衡需求 | 25%-40% | 平衡精确和召回 |
| 高召回需求 | 10%-25% | 尽量找出所有目标 |

### Q5：模板匹配和 YOLO 有什么区别？

| 对比项 | 模板匹配 | YOLO |
|--------|----------|------|
| 工作原理 | 像素级图像相似度 | 深度学习特征提取 |
| 速度 | 快 | 取决于模型大小 |
| 精度 | 对模板质量依赖高 | 泛化能力强 |
| 适用场景 | 固定界面元素 | 变化多样的目标 |
| 训练需求 | 无需训练 | 需要训练数据 |

### Q6：推理历史会占用多少空间？

**说明**：历史记录存储在浏览器本地存储中。建议定期清理不需要的记录以释放空间。

## 八、置信度与检测效果

### 8.1 置信度定义

置信度（Confidence）是模型对检测结果的确定性评估，范围 0-1：
- 0.9 = 90% 确信
- 0.5 = 50% 确信
- 0.1 = 10% 确信

### 8.2 置信度与精确度/召回率

| 置信度设置 | 精确度 | 召回率 | 说明 |
|------------|--------|--------|------|
| 高（>0.7） | 高 | 低 | 误检少，可能漏检 |
| 中（0.3-0.7） | 平衡 | 平衡 | 平衡结果 |
| 低（<0.3） | 低 | 高 | 误检多，不易漏检 |

### 8.3 来源标签说明

| 标签 | 颜色 | 含义 |
|------|------|------|
| YOLO | 橙色 | 由深度学习模型检测 |
| 模板 | 青色 | 由模板匹配检测 |

## 九、数据模型说明

推理测试涉及以下核心数据：

| 字段 | 说明 |
|------|------|
| id | 历史记录唯一标识 |
| name | 检测对象名称 |
| source_type | 来源类型（device/image/video） |
| inference_mode | 推理模式 |
| confidence_threshold | 使用的置信度阈值 |
| total_detections | 检测到的目标总数 |
| detections_json | 检测结果详情 |
| inference_time_ms | 推理耗时（毫秒） |
| created_at | 检测时间 |

设备检测数据：

| 字段 | 说明 |
|------|------|
| device_id | 设备唯一标识 |
| model | 设备型号 |
| resolution | 屏幕分辨率 |
