## 一、项目概述

### 1.1 项目背景

本项目旨在构建一套完整的游戏辅助系统，通过计算机视觉技术（YOLO模型）自动识别游戏画面，并模拟真人操作控制游戏。

### 1.2 项目目标

```
┌─────────────────────────────────────────────────────────────────────────┐
│                           项目目标                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│   阶段一：模型训练        阶段二：推理服务        阶段三：移动端App         │
│   ┌─────────────┐        ┌─────────────┐        ┌─────────────┐       │
│   │  训练平台   │ ──► ──►│  推理服务   │ ──► ──►│  Android App│       │
│   │  (已有)     │        │  (待开发)   │        │  (待开发)    │       │
│   └─────────────┘        └─────────────┘        └─────────────┘       │
│                                                                          │
│   用户可在平台上训练模型，并发布到推理服务，供移动端App调用                │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.3 系统特点

| 特点 | 说明 |
|------|------|
| **云端推理** | 模型部署在PC端/云服务器，移动端无需本地运行模型 |
| **反检测机制** | 服务端模拟真人操作行为，避免被游戏检测 |
| **多用户支持** | 一个服务端可服务多个移动端用户 |
| **模型可管理** | 用户可在平台训练、发布、更新模型 |
| **跨设备使用** | 只需一台可运行服务的PC/Mac，即可在任何手机上使用 |

#### 3.2.2 反检测技术实现

反检测技术用于模拟真人操作行为，避免被游戏检测系统识别为脚本。

**检测特征与应对策略：**

| 游戏检测特征 | 我们的解决方案 |
|-------------|---------------|
| 固定间隔操作 | 随机间隔 80-400ms |
| 精确坐标点击 | 坐标抖动 ±3像素 |
| 直线滑动轨迹 | 曲线+随机抖动 |
| 不知疲倦 | 随机暂停 0.5-2秒 |
| 100%准确 | 5%概率跳过动作 |

**时间间隔模拟器：**

```python
class HumanBehaviorSimulator:
    """模拟真人行为的时间间隔"""
    
    def __init__(self):
        self.base_interval_ms = 150
        
    def get_random_interval(self):
        # 使用正态分布模拟真人点击间隔
        interval = random.gauss(self.base_interval_ms, 50)
        
        # 10% 概率添加长停顿（真人会思考）
        if random.random() < 0.1:
            interval += random.randint(500, 2000)
        
        return max(50, min(interval, 3000))
```

**坐标抖动器：**

```python
class CoordinateJitter:
    """坐标抖动，模拟真人点击的轻微偏移"""
    
    def __init__(self, jitter_radius=3):
        self.jitter_radius = jitter_radius
        
    def add_jitter(self, x: int, y: int) -> tuple:
        jitter_x = random.randint(-self.jitter_radius, self.jitter_radius)
        jitter_y = random.randint(-self.jitter_radius, self.jitter_radius)
        return x + jitter_x, y + jitter_y
```

**滑动轨迹生成器：**

```python
class SwipeTrajectory:
    """生成自然的滑动轨迹"""
    
    def generate_natural_swipe(self, x1, y1, x2, y2, duration):
        trajectory = []
        steps = duration // 16  # 约60fps
        
        for step in range(steps + 1):
            t = step / steps
            
            # 基础线性插值
            x = x1 + (x2 - x1) * t
            y = y1 + (y2 - y1) * t
            
            # 添加曲线偏移
            curve_offset = math.sin(t * math.pi * 2) * 15
            
            # 添加随机抖动
            jitter_x = random.gauss(0, 2)
            jitter_y = random.gauss(0, 2)
            
            trajectory.append((int(x), int(y), int(duration * t)))
        
        return trajectory
```

## 六、部署方案

### 6.1 服务端部署

**Dockerfile：**

```dockerfile
FROM python:3.10-slim

WORKDIR /app

# 安装系统依赖
RUN apt-get update && apt-get install -y \
    libgl1-mesa-glx \
    libglib2.0-0 \
    && rm -rf /var/lib/apt/lists/*

# 复制依赖文件
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制应用代码
COPY . .

# 暴露端口
EXPOSE 8001

# 启动命令
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8001"]
```

**docker-compose.yml：**

```yaml
version: '3.8'

services:
  inference-service:
    build: .
    ports:
      - "8001:8001"
    volumes:
      - ./models:/app/models
      - ./logs:/app/logs
    environment:
      - MODEL_PATH=/app/models
      - LOG_LEVEL=info
    restart: unless-stopped
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: 1
              capabilities: [gpu]
```
