## 一、模块概述

模型管理模块用于管理训练完成的 YOLO 模型，支持模型的上传、查看、下载、激活等功能。模块提供详细的模型信息展示和训练效果评估曲线的可视化。

模块包含以下主要功能区域：

| 区域 | 功能说明 |
|------|----------|
| 统计概览 | 展示模型总数、活跃数、总大小等信息 |
| 模型列表 | 表格展示所有模型及其基本信息 |
| 模型详情 | 显示选中模型的完整信息 |
| 评估曲线 | 展示模型训练过程中的评估指标曲线 |
| 训练历史 | 展示模型的训练历史记录 |

---

## 二、支持的模型架构

系统支持以下 YOLO 模型架构：

| 架构 | 说明 | 适用场景 |
|------|------|----------|
| YOLOv8 Nano (yolov8n) | 最轻量，推理最快 | 移动设备、简单场景 |
| YOLOv8 Small (yolov8s) | 轻量级 | 资源受限环境 |
| YOLOv8 Medium (yolov8m) | 中等规模 | 平衡性能和精度 |
| YOLOv8 Large (yolov8l) | 大型，精度高 | 高精度需求 |
| YOLOv8 XLarge (yolov8x) | 超大型，最高精度 | 最佳效果 |
| YOLOv5 Nano (yolov5n) | YOLOv5 系列轻量版 | 兼容旧版本 |
| YOLOv5 Small (yolov5s) | YOLOv5 系列小版 | 兼容旧版本 |

## 四、各功能详细说明

### 4.1 统计概览

页面顶部显示系统的模型统计信息：

| 指标 | 说明 |
|------|------|
| 模型总数 | 当前系统中所有模型的数量 |
| 活跃模型 | 当前标记为活跃/默认的模型数量 |
| 总大小 | 所有模型文件占用的存储空间 |
| 架构分布 | 不同架构模型的数量分布 |

---

### 4.2 模型列表

**用途**：以表格形式展示所有已上传的模型。

#### 列表字段

| 字段 | 说明 |
|------|------|
| 名称 | 模型的文件名或指定名称 |
| 架构 | 模型的网络架构（如 yolov8n） |
| 状态 | 活跃/非活跃 |
| 性能 | mAP50 和 mAP50-95 指标 |
| 大小 | 模型文件的存储大小 |
| 上传时间 | 模型上传到系统的时间 |
| 操作 | 可执行的操作按钮 |

#### 状态说明

| 状态 | 含义 |
|------|------|
| 活跃（绿色） | 该模型被设为默认，当前推理使用此模型 |
| 非活跃（灰色） | 普通模型，非默认模型 |

#### 性能指标说明

| 指标 | 说明 | 含义 |
|------|------|------|
| mAP50 | 平均精度（IoU=50%） | IoU>50%时的检测精度 |
| mAP50-95 | 平均精度（IoU=50%-95%） | 多IoU阈值下的平均精度 |

#### 操作按钮

| 操作 | 说明 |
|------|------|
| 详情 | 查看模型的详细信息 |
| 设为默认 | 将该模型设为活跃的默认模型 |
| 删除 | 删除该模型（需确认） |

#### 筛选功能

点击列表右上角的架构筛选下拉框，可按模型架构进行筛选。

---

### 4.3 模型详情

**用途**：展示选中模型的完整参数和性能信息。

#### 显示内容

| 字段 | 说明 |
|------|------|
| 名称 | 模型名称 |
| 架构 | 模型的网络架构 |
| 格式 | 模型文件格式（如 .pt、.onnx） |
| 文件大小 | 模型文件的存储大小 |
| 训练参数 | epochs / batch_size / img_size |
| 性能指标 | mAP50-95 精度值 |
| 类别 | 模型检测的所有类别名称 |
| 描述 | 模型的详细说明 |
| 上传时间 | 模型上传时间 |

#### 评估曲线

如果模型关联了训练任务，详情页会显示训练过程中记录的评估曲线：

| 曲线类型 | 说明 |
|----------|------|
| Box Loss | 边界框回归损失，越低越好 |
| Cls Loss | 分类损失，越低越好 |
| mAP50 | IoU=50% 时的平均精度，越高越好 |
| mAP50-95 | 多IoU阈值的平均精度，越高越好 |

---

### 4.4 上传模型

**用途**：将本地训练好的模型文件上传到系统。

#### 支持的模型格式

| 格式 | 说明 |
|------|------|
| .pt | PyTorch 模型格式 |
| .pth | PyTorch 模型格式（另一种） |
| .onnx | ONNX 跨平台模型格式 |
| .tflite | TensorFlow Lite 移动端格式 |

#### 操作步骤

1. 点击页面右上角的「**上传模型**」按钮
2. 在弹出的上传窗口中填写信息：
   - **模型文件**：点击「选择模型文件」按钮，选择本地模型文件
   - **模型名称**：输入模型的名称，如 `game_ui_v1_best`
   - **架构**：选择模型的架构（yolov8n、yolov8s 等）
   - **描述**（可选）：模型的详细说明
3. 点击「**确定**」上传

---

### 4.5 设为默认模型

**用途**：将某个模型设为当前活跃的默认模型，推理时自动使用该模型。

#### 操作步骤

1. 在模型列表中找到目标模型
2. 点击该行的「**设为默认**」按钮
3. 按钮变为加载状态
4. 成功后，模型状态变为「活跃」

#### 说明

- 同一时间只能有一个活跃的默认模型
- 设置新默认模型后，之前的默认模型自动变为非活跃
- 设为默认后，该模型会显示在推理测试模块的模型选择中

---

### 4.6 训练历史

**用途**：查看模型的训练历史记录。

#### 操作步骤

1. 在模型详情页点击「**训练历史**」按钮
2. 在弹窗中查看该模型相关的训练任务列表

#### 历史记录内容

| 内容 | 说明 |
|------|------|
| 任务名称 | 训练任务的名称 |
| 架构 | 训练使用的模型架构 |
| 开始时间 | 训练开始的时间 |
| mAP50 | 训练完成后的 mAP50 指标 |
| mAP50-95 | 训练完成后的 mAP50-95 指标 |
| 状态 | 任务状态（已完成/进行中/失败） |

---

### 4.7 下载模型

**用途**：将模型文件下载到本地。

#### 操作步骤

1. 在模型详情页点击「**下载模型**」按钮
2. 浏览器开始下载模型文件
3. 下载完成后可在本地找到模型文件

#### 使用场景

- 部署到其他服务器
- 本地离线推理
- 备份模型文件

## 五、完整使用示例

### 场景：上传并管理一个新训练的模型

**目标**：将本地训练好的模型上传到系统，设为默认，并查看训练效果

#### Step 1：上传模型

1. 点击页面右上角「**上传模型**」按钮
2. 填写信息：
   - 模型文件：选择训练好的 `best.pt`
   - 模型名称：`game_ui_yolov8n_v1`
   - 架构：yolov8n
   - 描述：游戏界面检测 v1 模型
3. 点击确定

#### Step 2：查看模型详情

1. 在模型列表中点击「**详情**」按钮
2. 查看模型的各项参数：
   - 确认架构正确
   - 确认文件大小正常
   - 查看性能指标

#### Step 3：查看评估曲线

1. 在模型详情页向下滚动
2. 查看「评估曲线」区域的图表：
   - Box Loss 是否持续下降
   - mAP 指标是否持续上升
3. 曲线平稳说明训练正常

#### Step 4：设为默认模型

1. 点击「**设为默认**」按钮
2. 模型状态变为「活跃」
3. 现在推理测试会自动使用该模型

#### Step 5：下载备份

1. 点击「**下载模型**」按钮
2. 保存模型文件到本地备份目录

## 六、常见问题

### Q1：如何选择合适的模型架构？

**选择建议**：
- 资源受限（移动端、嵌入式）：选择 yolov8n
- 一般场景（桌面应用）：选择 yolov8s 或 yolov8m
- 高精度需求（服务器部署）：选择 yolov8l 或 yolov8x

### Q2：mAP 指标越高越好吗？

**说明**：是的，mAP（Mean Average Precision）是目标检测的标准指标，数值越高表示模型检测精度越好。

- mAP50 > 70%：较好
- mAP50 > 80%：优秀
- mAP50-95 > 50%：较好

### Q3：为什么有些模型没有评估曲线？

**原因**：
- 模型是手动上传的，未关联训练任务
- 训练任务未保存评估日志
- 训练任务进行中尚未完成

### Q4：可以删除默认模型吗？

**说明**：可以删除，但删除后需要手动将另一个模型设为默认。删除前请确认是否有其他模型可用。

### Q5：不同格式的模型有什么区别？

| 格式 | 特点 |
|------|------|
| .pt | PyTorch 格式，训练和微调首选 |
| .onnx | 跨平台，部署方便 |
| .tflite | 专为移动端优化 |

## 八、数据模型说明

模型管理涉及以下核心数据：

| 字段 | 说明 |
|------|------|
| id | 模型唯一标识 |
| name | 模型名称 |
| architecture | 模型架构（yolov8n 等） |
| format | 文件格式（pt/onnx/tflite） |
| file_size | 文件大小（字节） |
| is_active | 是否为默认模型 |
| description | 模型描述 |
| epochs | 训练轮数 |
| batch_size | 批大小 |
| img_size | 输入图片尺寸 |
| map50 | mAP50 性能指标 |
| map50_95 | mAP50-95 性能指标 |
| yolo_class_names | 检测类别名称列表 |
| training_job_id | 关联的训练任务 ID |
| created_at | 上传时间 |
