## 二、问题详情与修复记录

### Bug #1：数据集版本添加图片时未同步更新 class_ids

**问题描述**：

`dataset_version_crud.add_images()` 在向版本添加图片时，只更新了 `image_count`，但没有从新添加图片的标注中提取 `class_ids` 并合并到版本中。

这导致后续生成 YOLO 数据集时，`version.class_ids` 可能为空或不完整，进而导致部分类别的标注被忽略。

**影响范围**：

- `POST /datasets/{dataset_id}/versions/{version_id}/images`
- 影响 YOLO 数据集生成和模型训练

**修复方案**：

在 `add_images()` 方法中添加逻辑：
1. 从新增图片的标注中查询所有涉及到的 `class_id`
2. 与版本已有的 `class_ids` 合并
3. 更新版本的 `class_ids` 字段

**修复文件**：`GameAssistant/web/backend/app/crud/dataset.py`

**修复位置**：第 89-142 行

---

### Bug #2：YOLO 数据集生成时使用硬编码的类别名称

**问题描述**：

`generate_yolo_dataset()` 方法在生成 `data.yaml` 文件时，使用了硬编码的类别名称格式：

```yaml
names:
  - class_0
  - class_1
  - class_2
```

而不是从数据库中读取真实的类别名称（如 "player"、"enemy"、"coin"）。

**影响范围**：

- `POST /datasets/{dataset_id}/versions/{version_id}/generate-yolo`
- 影响训练时类别标签的可读性

**修复方案**：

1. 新增 `yolo_id_to_name` 字典，从 `AnnotationClass` 表中获取每个 YOLO ID 对应的真实类别名称
2. 使用真实名称生成 `names` 列表
3. 确保 YOLO ID 连续，生成完整的类别名称列表

**修复文件**：`GameAssistant/web/backend/app/crud/dataset.py`

**修复位置**：第 175-245 行

---

### Bug #3：训练启动时 YAML 默认内容格式错误

**问题描述**：

在 `_run_training()` 方法中，当 `version.dataset_yaml_content` 为空时，使用的默认 YAML 格式有误：

```yaml
path: .
tames:        # ← 错误：应该是 train
  - class_0
```

**影响范围**：

- 训练任务创建时（当数据集版本尚未生成 YOLO 格式时）
- 极少触发（因为训练前会检查 `yolo_dataset_path`）

**修复方案**：

将默认 YAML 修正为正确格式：

```yaml
path: .
train: train/images
val: val/images
names:
  - class_0
```

**修复文件**：`GameAssistant/web/backend/app/api/v1/training.py`

**修复位置**：第 96 行

---

### Bug #4：缺少训练完成后自动生成模型元数据的逻辑（建议增强）

**问题描述**：

虽然训练完成后会创建 `Model` 记录，但未保存训练时使用的 `class_ids` 和 `yolo_class_names`，导致后续推理时无法正确映射类别。

**修复方案**：

在 `_run_training()` 方法的训练成功分支中，补充获取并保存类别信息的逻辑：

1. 从 `DatasetVersion` 获取 `class_ids`
2. 查询对应的类别名称
3. 保存到 `Model` 记录的 `class_ids` 和 `yolo_class_names` 字段

**修复文件**：`GameAssistant/web/backend/app/api/v1/training.py`

**修复位置**：第 138-148 行

## 三、修复代码差异

### 3.1 `crud/dataset.py` - add_images 方法

```python
# 修复前：只更新 image_count
await db.execute(
    update(DatasetVersion)
    .where(DatasetVersion.id == version_id)
    .values(image_count=DatasetVersion.image_count + len(to_add))
)

# 修复后：同时更新 class_ids
new_class_ids = set(r[0] for r in ann_result.all() if r[0])
all_class_ids = list(existing_class_ids | new_class_ids)

await db.execute(
    update(DatasetVersion)
    .where(DatasetVersion.id == version_id)
    .values(
        image_count=DatasetVersion.image_count + len(to_add),
        class_ids=all_class_ids if all_class_ids else DatasetVersion.class_ids,
    )
)
```

### 3.2 `crud/dataset.py` - generate_yolo_dataset 方法

```python
# 修复前：硬编码类别名称
names_content = "\n".join(
    f"  - class_{i}" for i in range(len(class_map))
)

# 修复后：使用真实类别名称
class_id_to_yolo: dict[str, int] = {}
yolo_id_to_name: dict[int, str] = {}
for cid in class_ids:
    cls = await db.execute(
        select(AnnotationClass).where(AnnotationClass.id == cid)
    ).scalar_one_or_none()
    if cls:
        class_id_to_yolo[cls.id] = cls.yolo_class_id
        yolo_id_to_name[cls.yolo_class_id] = cls.name

max_yolo_id = max(yolo_id_to_name.keys(), default=-1)
names_list = [yolo_id_to_name.get(i, f"class_{i}") for i in range(max_yolo_id + 1)]
names_content = "\n".join(f"  - {name}" for name in names_list)
```

## 四、完整 YOLO 训练链路操作流程

修复后，完整的训练链路如下：

```
1. 图片管理
   ├── 上传图片（单张/批量 ZIP）
   └── 视频抽帧（间隔/数量/场景检测）

2. 标注管理
   ├── 创建标注项目
   ├── 添加图片到项目
   ├── 创建标注类别（设置 YOLO ID）
   ├── 在图片上绘制标注框
   └── 提交审核/审核通过

3. 数据集管理
   ├── 创建数据集
   ├── 创建数据集版本
   ├── 向版本添加图片（自动同步 class_ids）
   └── 生成 YOLO 格式数据集

4. 训练管理
   ├── 选择数据集版本
   ├── 配置训练参数
   ├── 发起训练任务
   ├── 实时监控进度和指标
   └── 训练完成后自动生成模型

5. 模型管理
   ├── 查看训练生成的模型
   ├── 设为默认模型
   └── 下载模型文件

6. 推理测试
   ├── 设备实时推理（ADB）
   ├── 图片批量推理
   └── 视频推理
```

## 六、注意事项

### 6.1 训练前提条件

1. **GPU 环境**：必须安装 NVIDIA GPU 和 CUDA
2. **ultralytics 库**：确保已安装 `pip install ultralytics`
3. **数据集就绪**：训练前必须先执行"生成 YOLO 数据集"操作

### 6.2 类别 ID 规范

- 每个类别的 `yolo_class_id` 必须唯一且从 0 开始
- 建议按顺序创建类别，避免 ID 不连续
- 同一数据集版本中所有类别 ID 应保持稳定

### 6.3 训练参数建议

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| epochs | 50-300 | 训练轮数 |
| batch_size | 8/16/32 | 根据显存调整 |
| img_size | 640/1280 | 输入图片尺寸 |
| patience | 50 | 早停耐心值 |
