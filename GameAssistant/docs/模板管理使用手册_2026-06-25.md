## 一、模块概述

模板管理模块用于管理图像模板匹配所需的模板图片。这些模板图片是游戏辅助系统中用于识别游戏界面元素（如按钮、图标等）的参考图片，通过模板匹配算法在屏幕画面中查找相似的区域。

模块包含以下主要功能区域：

| 区域 | 功能说明 |
|------|----------|
| 模板目录 | 以树形结构展示所有模板，按类别分组 |
| 模板列表 | 网格展示所有模板卡片，支持搜索和筛选 |
| 模板预览 | 显示选中模板的大图预览 |
| 模板详情 | 展示模板的详细信息和参数 |
| 匹配参数 | 配置模板匹配的测试参数 |

## 三、各功能详细说明

### 3.1 上传模板

**用途**：将新的模板图片上传到系统。

#### 操作步骤

1. 点击页面右上角的「**上传模板**」按钮
2. 在弹出的上传窗口中填写信息：
   - **模板文件**：点击「选择 PNG/JPG 图片」按钮，选择本地图片文件（支持 .png、.jpg、.jpeg、.bmp、.webp 格式）
   - **类别**：从下拉列表中选择模板所属的类别（类别来源于标注管理中的类别定义）
   - **模板名称**：输入模板的名称，如 `btn_attack_normal.png`
   - **匹配阈值**：拖动滑块设置模板匹配的相似度阈值（默认 85%）
3. 点击「**确定**」完成上传

#### 匹配阈值说明

| 阈值范围 | 匹配效果 | 适用场景 |
|----------|----------|----------|
| 90%-100% | 严格模式 | 精确匹配，可能漏检 |
| 75%-90% | 平衡模式（推荐） | 兼顾准确性和召回率 |
| 50%-75% | 宽松模式 | 高召回率，可能误检 |

---

### 3.2 模板目录（左侧树形结构）

**用途**：按类别分组浏览所有模板。

#### 功能特点

- 目录以树形结构展示，根节点为类别名称
- 每个类别节点显示该类别下的模板数量
- 点击类别名称可以筛选该类别的所有模板
- 点击具体模板名称可以选中该模板

#### 操作说明

| 操作 | 说明 |
|------|------|
| 点击类别 | 筛选显示该类别下的所有模板 |
| 点击模板名 | 选中该模板，在右侧显示详情 |

---

### 3.3 模板列表（网格视图）

**用途**：以卡片网格形式展示所有模板。

#### 功能特点

- 每个模板显示为一张卡片，包含缩略图、名称、类别标签
- 支持搜索功能，可按模板名称搜索
- 支持按类别筛选
- 懒加载优化，性能更好

#### 卡片操作

| 操作 | 说明 |
|------|------|
| 点击卡片 | 选中模板，显示详情 |
| 点击开关 | 启用/禁用该模板 |
| 点击删除 | 删除该模板（需确认） |

#### 状态说明

| 状态 | 含义 |
|------|------|
| 启用 | 模板可参与匹配识别 |
| 禁用 | 模板暂时不参与匹配 |

---

### 3.4 模板预览

**用途**：查看选中模板的大图预览。

#### 功能说明

- 显示当前选中模板的高清图片
- 鼠标悬停时显示「点击放大」提示
- 点击可打开全尺寸的弹窗预览

---

### 3.5 模板详情

**用途**：展示选中模板的完整参数信息。

#### 显示内容

| 字段 | 说明 |
|------|------|
| 名称 | 模板的文件名 |
| 类别 | 所属的检测类别 |
| 尺寸 | 图片的宽 x 高（像素） |
| 匹配阈值 | 该模板的匹配相似度要求 |
| 状态 | 当前是否启用 |
| 上传时间 | 模板上传到系统的时间 |

---

### 3.6 测试匹配

**用途**：在指定图片上测试模板的匹配效果。

#### 操作步骤

1. **选择图片**：从下拉列表中选择一张已上传的测试图片
2. **选择模板**（可选）：选择要测试的特定模板，留空则测试全部模板
3. **设置阈值**：调整匹配阈值滑块
4. **开启多匹配**：如果需要匹配多个结果，开启「多匹配」开关
5. **选择匹配方法**：
   - 归一化相关系数（推荐）：最常用的匹配方法
   - 平方差匹配：适合光照变化场景
   - 相关系数：快速匹配
   - 平方差归一化：高精度匹配
6. 点击「**测试匹配**」按钮查看结果

#### 匹配方法对比

| 方法 | 适用场景 | 速度 |
|------|----------|------|
| tm_ccoeff（归一化相关系数） | 通用场景，推荐使用 | 中等 |
| tm_sqdiff（平方差） | 光照变化 | 快 |
| tm_ccoeff（相关系数） | 快速匹配 | 快 |
| sqdiff_normed（平方差归一化） | 高精度需求 | 慢 |

#### 结果解读

| 状态 | 含义 |
|------|------|
| 匹配 | 模板在图片中被找到，显示匹配度百分比和位置坐标 |
| 未匹配 | 模板在图片中未找到或相似度低于阈值 |

## 四、完整使用示例

### 场景：为游戏按钮创建模板

**目标**：创建游戏攻击按钮的模板，用于后续的模板匹配识别

#### Step 1：准备模板图片

1. 在游戏中截取攻击按钮的清晰截图
2. 使用图片编辑工具裁剪，只保留按钮区域
3. 保存为 PNG 格式，分辨率建议 32x32 到 128x128 之间

#### Step 2：上传模板

1. 进入「模板管理」页面
2. 点击右上角「**上传模板**」按钮
3. 填写信息：
   - 模板文件：选择准备好的按钮图片
   - 类别：选择或创建「攻击按钮」类别
   - 模板名称：`btn_attack_normal.png`
   - 匹配阈值：85%
4. 点击确定

#### Step 3：验证匹配效果

1. 在「测试匹配」区域选择一张包含攻击按钮的游戏截图
2. 确认选择的是刚上传的模板
3. 点击「**测试匹配**」
4. 检查结果：
   - 如果显示「匹配」且位置正确，说明模板可用
   - 如果显示「未匹配」，尝试降低阈值或更换图片

#### Step 4：启用模板

1. 在模板列表中找到刚上传的模板
2. 确保启用开关处于开启状态
3. 模板即可在推理测试中使用

## 五、常见问题

### Q1：为什么模板匹配总是失败？

**可能原因**：
1. 模板图片与实际画面存在差异（缩放、旋转、形变）
2. 光照条件不同
3. 阈值设置过高

**解决方法**：
1. 确保模板图片是在实际游戏环境中截取的
2. 降低匹配阈值（建议从 85% 开始尝试）
3. 尝试不同的匹配方法

### Q2：如何提高匹配准确性？

**建议**：
1. 模板图片应尽量干净，背景简单
2. 避免在模板中包含太多无关内容
3. 考虑创建多个变体模板应对不同场景
4. 对于光照变化，尝试使用「平方差匹配」方法

### Q3：模板启用/禁用的区别是什么？

**说明**：
- **启用**：该模板会参与推理时的模板匹配
- **禁用**：该模板暂时不参与匹配，可在不删除的情况下临时排除

### Q4：一个类别可以有多少个模板？

**说明**：一个类别可以上传多个模板图片，这在以下场景很有用：
- 同一个按钮的不同状态（正常/高亮/按下）
- 同一个按钮的不同样式变体
- 不同分辨率下的模板

## 七、数据模型说明

模板管理涉及以下核心数据：

| 字段 | 说明 |
|------|------|
| id | 模板唯一标识 |
| name | 模板名称 |
| class_name | 所属类别 |
| width | 模板宽度（像素） |
| height | 模板高度（像素） |
| match_threshold | 匹配阈值 |
| is_active | 是否启用 |
| created_at | 创建时间 |
