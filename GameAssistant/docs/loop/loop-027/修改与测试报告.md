### 1.4 技术要点说明

#### Vite 分包策略优化
本次优化针对 669KB index chunk 问题：

**优化前问题**：
- antd 组件通过 unplugin-vue-components 自动导入后，Rollup 无法有效拆分
- 导致大量小 index chunk 分散

**优化后策略**：
1. **移除 antd 手动分包**：让 Vite/Rollup 自然处理组件库的代码分割
2. **保留 echarts 分包**：ECharts 体积大（520KB），单独分包便于缓存
3. **仅分包 vue-vendor + axios**：核心库单独分包

**构建结果对比**：
| 指标 | 优化前 | 优化后 | 变化 |
|---|---|---|---|
| 最大 JS chunk | 669.64 KB | 174.94 KB | -74% |
| index chunks 数量 | 20+ 个 | 更少、更合理 | 改善 |
| 构建时间 | ~10s | ~9s | 略快 |

## 四、备注

### 4.1 关于 669KB chunk 的分析
通过分析构建输出，确认最大 chunk 来源为 ant-design-vue 库的完整代码。Vite 的 tree-shaking 对 UI 组件库效果有限，因为：
1. 组件库内部模块相互依赖
2. CSS-in-JS 样式代码无法被 tree-shake

**可行的进一步优化方向**（P1）：
1. 使用 `unplugin-vue-components` 的 `resolve` 选项配合 `importStyle: 'less'` + Less 变量主题
2. 将更多非首屏组件改为真正的动态导入（配合路由懒加载）
3. 考虑使用 `vite-plugin-compression` 添加 gzip/brotli 压缩
