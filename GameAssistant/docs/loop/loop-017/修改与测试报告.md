## 二、本次实现的 Bug 修复

### 2.1 API 路径 double v1 Bug

**问题**：`baseURL: '/api/v1'` + Vite proxy `rewrite: /api → /api/v1` 导致所有请求路径变成 `/api/v1/v1/...`

**修复方案**：
- `request.ts` baseURL 改为 `/api`
- Vite proxy 保持 `/api` → `/api/v1` 的 rewrite
- 各 API 文件中的 4 个硬编码 `/api/v1/...` URL 改为 `/api/...`

**修复后效果**：前端请求 `/api/xxx` → Vite proxy 重写为 `/api/v1/xxx` → 后端接收正确
