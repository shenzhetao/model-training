### 1.4 技术要点说明

#### 后端实现
1. **用户模型扩展**：新增 `is_password_changed` 字段，用于强制用户首次登录后修改密码
2. **Admin API 端点**：
   - `GET /api/v1/users/` - 列出所有用户（需要 admin 权限）
   - `GET /api/v1/users/{user_id}` - 获取用户详情（需要 admin 权限）
   - `POST /api/v1/users/` - 创建用户（需要 admin 权限）
   - `PUT /api/v1/users/{user_id}` - 更新用户（需要 admin 权限）
   - `DELETE /api/v1/users/{user_id}` - 删除用户（需要 admin 权限）
3. **权限控制**：使用 `require_role("admin")` 依赖确保只有管理员可以访问用户管理 API

#### 前端实现
1. **用户管理页面**：展示用户列表、创建用户、编辑用户、删除用户功能
2. **管理员菜单**：仅 admin 用户可见"用户管理"菜单项
3. **路由守卫**：`requiresAdmin` meta 标记保护用户管理路由
4. **登录优化**：登录后立即获取并存储用户信息到 Pinia store
