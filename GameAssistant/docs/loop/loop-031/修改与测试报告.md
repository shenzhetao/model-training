## 一、本次修改内容

### 1.1 修复 Modal 弹窗选择器问题

**问题描述**: E2E 测试中使用 `getByRole('heading')` 选择器无法正确匹配 Modal 弹窗标题

**修改文件**:
- `e2e/tests/images.spec.ts` - 页面标题选择器
- `e2e/tests/datasets.spec.ts` - 页面标题选择器
- `e2e/tests/models.spec.ts` - 页面标题选择器
- `e2e/tests/templates.spec.ts` - 页面标题选择器
- `e2e/tests/training-basic.spec.ts` - 页面标题和弹窗选择器
- `e2e/tests/inference.spec.ts` - 页面标题选择器

**修改内容**:
```typescript
// 修改前
await expect(page.getByRole('heading', { name: /图片管理/i })).toBeVisible()

// 修改后
await expect(page.locator('h2').filter({ hasText: /图片管理/i })).toBeVisible()
```

### 1.2 修复 chromium-fresh 测试配置

**问题描述**: chromium-fresh 项目与需要 auth 的测试页面不兼容

**修改文件**: `e2e/playwright.config.ts`

**修改内容**:
- 为 chromium-fresh 项目添加 `testMatch` 限制为只运行 `login.spec.ts`
- 添加 `testIgnore` 排除所有需要 auth 的测试文件

```typescript
{
  name: 'chromium-fresh',
  testMatch: ['**/login.spec.ts'],
  testIgnore: ['**/page-load.spec.ts', '**/training*.spec.ts', ...],
  use: { ...devices['Desktop Chrome'] }
}
```

### 1.3 修复训练管理弹窗测试

**问题描述**: Modal 标题选择器 `.or()` 导致 strict mode violation

**修改文件**: `e2e/tests/training-basic.spec.ts`

**修改内容**:
- 移除 `.or()` 链式调用，改用 `.first()` 优先选择器
- 使用 `Escape` 键关闭弹窗替代点击取消按钮

```typescript
// 修改前
await expect(page.locator('.ant-modal .ant-modal-title').or(page.locator('.ant-modal-header')).first()).toBeVisible()

// 修改后
await expect(page.locator('.ant-modal .ant-modal-title').first()).toBeVisible()
```

### 1.4 修复其他弹窗关闭逻辑

**修改文件**:
- `e2e/tests/datasets.spec.ts` - 使用 Escape 键关闭弹窗
- `e2e/tests/models.spec.ts` - 使用 Escape 键关闭弹窗
