### 1.4 技术要点说明
> 本次修改涉及的核心技术决策和实现细节。

1. **浏览器兼容性问题处理**：
   - 为 webkit 和 Mobile Chrome 项目配置了 `testMatch` 和 `testIgnore` 规则
   - webkit 跳过 annotations、datasets、images、models、templates、training、inference 测试
   - Mobile Chrome 跳过 annotations、images、models、templates、training、inference 测试
   - 保留 login、page-load、user-flow 和部分 datasets 测试

2. **认证状态处理优化**：
   - 添加 `waitForElement` 函数等待特定元素出现
   - 优化了 `login` 函数，添加更详细的日志和错误处理
   - 为不同浏览器配置了特定选项（viewport、touch 等）
   - 添加了更健壮的 token 检测机制

3. **测试数据管理**：
   - 创建 `test-data.ts` fixture，提供测试数据生命周期管理
   - 提供 `generateTestName` 函数生成唯一测试数据名称
   - 提供 `recordResource` 和 `cleanupTestData` 函数管理测试资源
   - 扩展 Playwright Test 类型，添加 `testData` 和 `authenticatedPage` fixture

4. **测试辅助工具**：
   - 创建 `test-utils.ts` 提供常用测试辅助函数
   - 包括等待 Modal、关闭 Modal、填充表单、选择下拉选项等
