### 1.2 技术要点说明

#### 问题分析
Firefox/WebKit 认证失败（403 错误）的根本原因是 `global-setup.ts` 中的登录等待逻辑不够健壮：

1. **原代码问题**：
   - 只使用 `waitForURL` 等待跳转，但没有验证 localStorage 中的 token 是否真正写入
   - 缺少对网络请求完成（`networkidle`）的等待
   - 没有验证登录后是否真的离开了登录页面

2. **跨浏览器差异**：
   - Firefox 和 WebKit 对 Vue.js 和 localStorage 的处理时序可能与 Chromium 不同
   - Pinia store 更新和 Vue 响应式更新可能有延迟
   - 网络请求完成时间在浏览器间可能有差异

#### 修复方案

1. **增强登录流程**：
   - 使用 `waitUntil: 'networkidle'` 等待登录页加载完成
   - 使用 `waitForLoadState('domcontentloaded')` 确保 DOM 准备好
   - 点击提交后使用 `Promise.all` 并行等待导航和点击完成

2. **关键改进 - 轮询等待 localStorage**：
   ```typescript
   await page.waitForFunction(() => {
     const token = localStorage.getItem('gameassistant_token')
     return token !== null && token !== ''
   }, { timeout: 15000 })
   ```
   这确保了 token 被写入后才继续，而不是仅仅依赖 URL 变化。

3. **增加额外等待**：
   - 登录成功后额外等待 1 秒，确保 Vue/Pinia 完全处理 token
   - 验证最终 URL 确认不在登录页面

4. **详细日志**：
   - 添加登录流程各阶段的日志，便于调试

## 四、备注

### 4.1 可能的遗留问题

1. **Token 过期时间**：JWT token 有效期为 24 小时，测试认证文件可能需要定期刷新
2. **浏览器未安装**：如果 Firefox/WebKit 未安装，测试会自动跳过并提示安装
3. **并发测试**：多个浏览器同时运行可能造成资源竞争，建议串行执行

### 4.2 后续优化建议

1. 考虑使用 cookie 而非 localStorage 存储 JWT（cookie 配合 httpOnly 更安全）
2. 添加 token 刷新机制
3. 为每个浏览器项目添加独立的超时配置
