### 主要修复

1. **TypeScript 编译错误（16 个）**
   - 修复未使用的导入（`modelsApi`, `PlusOutlined`, `Empty`）
   - 修复未使用的变量（`offsetX`, `offsetY`, `annotationColumns`, `loading`, `index`）
   - 修复 `Empty.PRESENTED_IMAGE_SIMPLE` 在模板中无法解析的问题
   - 修复 `TemplateManager.testMatching` 缺少 `multi_match` 和 `method` 参数
   - 修复 `ModelManager` 死代码（`addTag`, `removeTag`）

2. **Vue 模板解析错误**
   - 修复 `<a-divider>text</a-divider>` 改为自闭合 + 分离标签
   - 修复 `title="已完成项目"` 中 `>` 导致模板解析错误
   - 修复 `record.total_images > 0` 在 `:percent` 属性中导致模板解析错误
   - 修复 `<a-empty>` v-else 位置不正确
   - 添加缺失的 `</style>` 标签

3. **Docker 服务修复**
   - MySQL 端口从 3306 改为 3307（避免与宿主机冲突）
   - 移除 docker-compose.yml 中过时的 `version` 字段
   - `opencv-python` 改为 `opencv-python-headless`（避免 X11 依赖）

4. **后端依赖兼容性**
   - `aiomysql==0.2.0` + SQLAlchemy 2.x `pool_pre_ping` 兼容性问题，禁用 `pool_pre_ping`
   - `passlib` + `bcrypt 5.x` 不兼容，移除 passlib 改用原生 `bcrypt`
   - `CORS_ORIGINS` 环境变量解析错误，改为 `@property` 方法

5. **Wrapper 模块降级处理**
   - `detector_wrapper.py` 和 `template_wrapper.py` 中 `template_matcher` 导入改为 try/except，缺失时优雅降级
   - 避免因外部模块不可用导致整个后端启动失败

6. **Nginx API 路由**
   - 添加 `/api/v1` 专用路由块，解决 `baseURL: /api` 与后端 `/api/v1` 前缀不匹配问题
   - 保持与 Vite dev proxy `rewrite: /api -> /api/v1` 的一致性

7. **登录 API 修复**
   - `Login.vue` 从 `/auth/login` 改为 `/users/login/json`（支持 JSON body）
   - `authStore.setAuth` 参数类型改为 `User | null`
   - 数据库初始化 admin 用户（bcrypt hash 写入）
