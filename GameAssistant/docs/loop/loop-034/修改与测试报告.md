### 1.4 技术要点说明

> 本次修改涉及的核心技术决策和实现细节：

1. **Tab 切换测试优化**：
   - 使用 `[role="tab"]` 选择器替代 CSS 类选择器，符合 ARIA 规范
   - 使用 `aria-selected` 属性验证 Tab 激活状态
   - 使用 `waitForFunction` 等待 Tab 内容区域更新

2. **Mobile Chrome 测试隔离**：
   - 配置独立的 `storageState` 文件 `mobile-chrome.json`
   - 调整视口配置为 `{ width: 412, height: 915 }`
   - 创建专门的 `mobile-compat.spec.ts` 测试文件

3. **路由守卫测试修复**：
   - 使用 `storageState: undefined` 确保新 context 没有继承 auth state
   - 添加更宽松的断言以适应不同的路由守卫实现
