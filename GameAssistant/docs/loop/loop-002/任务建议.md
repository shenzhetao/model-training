### 1.2 质量问题

#### 🔴 必须修复的 Bug

**1. 语法错误 - `web/frontend/src/api/images.ts` 第 1 行**

```typescript
// 当前（错误）：
import request from './request'

// 应为：
import request from './request'
```
> ⚠️ 第 1 行存在语法错误，缺少关键字 `import` 后面的变量名。实际文件内容显示第 1 行为空，错误应在第 2 行附近。需要检查实际文件确认。

**2. API URL 拼接错误 - `web/frontend/src/api/images.ts` 第 73 行**

```typescript
// 当前：
return request.get(`/images/${query ? `?${query}` : ''}`)

// 问题：request 基础 URL 已是 `/api/v1`，再拼接 `/images/` 会导致
//       最终 URL 变成 `/api/v1/images/?page=1` 但请求路径实际是 `/api/v1/api/v1/images/`

// 应为：
return request.get(`/images${query ? `?${query}` : ''}`)
```

#### 🟡 代码质量问题

**3. 前端 `request.ts` 类型覆盖不完整**

```typescript
// 当前第 73-85 行：手动覆盖了 post/delete/get，但 put/patch 未覆盖
// 若其他地方调用 request.put() 会使用原始 axios 实例，跳过拦截器
```

**4. ZIP 批量上传未使用后台任务**

```python
# 当前 images.py 第 168 行：
background_tasks: BackgroundTasks = None,

# 问题：BackgroundTasks 参数声明了但未使用，大批量上传仍会阻塞请求
# 应使用 background_tasks.add_task() 将上传任务放入后台
```

**5. 数据库事务边界问题**

```python

db_image = await image_crud.create(db, image_data)
await db.commit()
return db_image

# 问题：在路由处理函数中直接 commit，应在 CRUD 层统一管理事务
# 建议：移除路由中的 commit，改为依赖依赖注入的 session 或统一事务管理
```

**6. 批量删除权限检查不完整**

```python
# images.py 第 476-477 行：

    continue  # 静默跳过无权限的图片

# 问题：静默跳过会让用户困惑，不知道哪些删除失败了
# 建议：记录无权限的图片 ID，在响应中返回
```

#### 🟢 架构建议

**7. 缺少 `serveUrl` 去重逻辑**

根据修改报告，`getServeUrl` 方法在第 27 行做了去重处理：
```typescript
getServeUrl(id: string): string {
  let url = `/images/${id}/serve`
  return url.startsWith('/api/v1') ? url : `/api/v1${url}`
}
```
> 需确认实际文件是否已包含此逻辑

**8. 缺少 `uploaded_by` 关联查询**

当前 `get_multi_paginated` 支持 `uploaded_by` 过滤，但前端未提供按用户筛选功能（仅管理员需要）

## 三、风险提示

### 🔴 高风险

**1. 前后端联调风险**
- API URL 拼接 bug（已发现）可能导致前端无法加载图片列表
- 需要尽快验证并修复

**2. JWT 认证依赖**
- 所有图片 API 都依赖 `get_current_user` 认证
- 如果认证流程有问题，整个模块无法使用
- 建议在 loop-003 开始前先验证登录功能

### 🟡 中风险

**3. 数据库迁移缺失**
- `source_videos`、`video_extraction_tasks` 表的 Flyway 迁移脚本未创建
- V1__initial_schema.sql 可能不包含这些表
- 视频抽帧功能依赖这些表

**4. ADB 环境依赖**
- 平台运行机器需要安装 `adb` 命令行工具
- 需要检测 `adb` 是否可用并给出友好提示

**5. 存储空间风险**
- 当前所有文件存储在 `uploads/` 目录
- 大规模使用时需考虑磁盘空间监控
- 建议添加存储使用量统计

### 🟢 低风险

**6. 批量操作性能**
- 批量删除 1000 张图片时逐个查询权限
- 建议后续优化为批量查询权限

**7. 图片安全**
- 当前仅检查文件扩展名
- 建议添加文件内容 magic bytes 检查
- 防止伪装的恶意文件
