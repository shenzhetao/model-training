### 1.3 技术要点说明

1. **图片上传**：
   - 单张上传：`POST /api/v1/images/upload`，支持文件大小 50MB 限制
   - ZIP 批量上传：`POST /api/v1/images/upload/batch`，支持 2GB ZIP 文件
   - 上传前检查 MD5 去重，已存在返回 409 Conflict
   - 使用 UUID 重命名文件存储，保留原始文件名在数据库

2. **图片存储**：
   - 存储目录：`uploads/images/`
   - 文件命名：`{uuid}.{ext}`
   - 静态文件通过 `/uploads` 路径访问

3. **MD5 去重**：
   - 上传前计算文件 MD5 哈希
   - 查询数据库是否存在相同 MD5 且未删除的图片
   - 已存在返回 `is_duplicate: true` 和 `existing_image_id`

4. **软删除**：
   - 删除操作设置 `is_deleted=True`，不物理删除数据库记录
   - 批量删除支持最多 1000 张

5. **流式读取**：
   - `GET /api/v1/images/{id}/serve` 流式返回图片文件
   - 设置 `Cache-Control: public, max-age=31536000` 缓存一年
   - 根据文件扩展名设置正确的 Content-Type

6. **前端灯箱**：
   - 支持键盘左右箭头翻页
   - ESC 键关闭
   - 显示图片文件名、尺寸、大小、当前位置

7. **拖拽上传**：
   - 拖拽区域支持拖放图片或 ZIP 文件
   - 自动识别文件类型
