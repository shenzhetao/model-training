> 文档版本：v1.0

## 一、模块概述

训练管理模块是平台的核心功能之一，用于启动和管理 YOLO 模型的训练任务。模块提供完整的训练参数配置、实时进度监控、训练指标可视化等功能。

模块包含以下主要功能区域：

| 区域 | 功能说明 |
|------|----------|
| 训练任务列表 | 展示所有训练任务及其状态 |
| 任务详情 | 显示选中任务的详细参数 |
| 训练指标图表 | 可视化展示训练过程中的各项指标 |
| 原始日志 | 显示训练过程的原始输出日志 |

## 二、支持的训练参数

### 2.1 模型架构

| 架构 | 说明 | 特点 |
|------|------|------|
| YOLOv8 Nano | 最轻量级 | 推理最快，精度相对较低 |
| YOLOv8 Small | 轻量级 | 平衡速度和精度 |
| YOLOv8 Medium | 中等规模 | 更高的精度 |
| YOLOv8 Large | 大型 | 高精度，资源消耗大 |
| YOLOv8 XLarge | 超大型 | 最高精度，需要更多资源 |

### 2.2 核心训练参数

| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| Epochs | 1-1000 | 50 | 训练轮数，整个数据集的训练次数 |
| Batch Size | 1-256 | 8 | 每批处理的图片数量 |
| 图片尺寸 | 320-1280 | 640 | 输入网络的图片大小（像素） |
| 初始学习率 | 0-1 | 0.01 | 优化器的初始学习率 |
| 最终学习率 | 0-1 | 0.01 | 训练结束时的学习率 |
| Early Stop | 5-200 | 15 | 早停轮数，无改进时停止训练 |

### 2.3 数据增强参数

| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| Mosaic | 0-1 | 1.0 | 马赛克增强概率（将4张图拼接） |
| MixUp | 0-1 | 0.0 | MixUp 增强概率（混合两张图） |
| 水平翻转 | 0-1 | 0.5 | 随机水平翻转的概率 |

## 四、各功能详细说明

### 4.1 训练任务列表

**用途**：展示所有训练任务，支持状态筛选和任务选择。

#### 列表字段

| 字段 | 说明 |
|------|------|
| 任务名称 | 训练任务的名称 |
| 架构 | 使用的模型架构 |
| Epochs | 训练轮数（当前/总） |
| 状态 | 任务当前状态 |
| 进度 | 训练进度百分比 |
| mAP | 最终 mAP50 性能指标 |
| 创建时间 | 任务创建时间 |
| 操作 | 可执行的操作 |

#### 状态说明

| 状态 | 颜色 | 含义 |
|------|------|------|
| 等待 | 灰色 | 任务等待调度执行 |
| 进行 | 蓝色 | 正在训练中 |
| 完成 | 绿色 | 训练已完成 |
| 失败 | 红色 | 训练过程中出错 |
| 已停止 | 橙色 | 训练被手动停止 |

#### 状态筛选

点击列表右上角的下拉框，可按状态筛选任务：
- 全部
- 等待中
- 进行中
- 已完成
- 失败

---

### 4.2 新建训练任务

**用途**：创建一个新的训练任务。

#### 操作步骤

1. 点击页面右上角的「**新建训练**」按钮
2. 填写训练任务信息：

**基本配置**

| 字段 | 说明 | 示例 |
|------|------|------|
| 任务名称 | 给训练任务起个名字 | game_ui_yolov8n_v1 |
| 数据集版本 | 选择要使用的数据集版本 | game_ui_v1 / baseline |
| 模型架构 | 选择基础模型架构 | yolov8n（最轻量） |

**训练参数**

| 参数 | 说明 | 建议 |
|------|------|------|
| Epochs | 训练轮数 | 50-200，新项目建议 100+ |
| Batch Size | 批大小，根据显存调整 | 8-32，显存 8G 建议 16 |
| 图片尺寸 | 输入图片大小 | 640，小目标可 320 或 1280 |
| 初始学习率 | 优化器起始学习率 | 0.01 |
| 最终学习率 | 训练结束学习率 | 0.01 |
| Early Stop | 无改进时停止的轮数 | 15-50 |

**数据增强参数**

| 参数 | 说明 | 建议 |
|------|------|------|
| Mosaic | 马赛克增强，1.0=全部使用 | 默认 1.0，效果好 |
| MixUp | MixUp 增强，0.0=不使用 | 默认 0.0，防止过拟合 |
| 水平翻转 | 随机水平翻转概率 | 默认 0.5 |

3. 点击「**确定**」开始训练

#### 参数调整建议

| 场景 | Epochs | Batch Size | 图片尺寸 |
|------|--------|------------|----------|
| 快速测试 | 10-20 | 16 | 320 |
| 正常训练 | 50-100 | 8-16 | 640 |
| 高精度 | 200-500 | 4-8 | 1280 |

### 4.4 训练指标图表

**用途**：可视化展示训练过程中的各项指标。

#### 图表标签页

**Loss 图表**

展示各类损失的变化趋势：
- Box Loss：边界框回归损失
- Cls Loss：分类损失
- Val Box：验证集边界框损失

**mAP 图表**

展示精度指标的变化趋势：
- mAP50：IoU=50% 时的平均精度
- mAP50-95：多 IoU 阈值的平均精度

**Precision/Recall 图表**

展示查准率和查全率：
- Precision：预测框的准确率
- Recall：目标被检出的比例

#### 图表解读

| 曲线走势 | 含义 |
|----------|------|
| 持续下降（Loss） | 训练正常，模型在学习 |
| 持续上升 | 异常，可能需要调整参数 |
| 剧烈波动 | 学习率可能过高 |
| 趋于平稳 | 模型接近收敛 |
| 持续上升后下降（验证Loss） | 可能过拟合 |

### 4.5 原始日志

**用途**：查看训练过程的原始输出日志。

#### 内容说明

日志区域显示训练过程中的原始输出，包括：
- 每个 Epoch 的详细指标
- 损失值变化
- mAP 计算结果
- 警告和错误信息

#### 实时更新

训练进行中时，日志区域会自动滚动更新最新的训练输出。

#### 下载日志

点击「**下载日志**」按钮可导出训练日志到本地文件，便于离线分析。

---

### 4.6 停止训练

**用途**：手动停止正在进行的训练任务。

#### 操作步骤

1. 在任务列表中找到正在训练的任务
2. 点击该行的「**停止**」按钮
3. 任务状态变为「已停止」

#### 说明

- 停止后，已训练的 Epoch 数据会保留
- 可以查看已完成的训练效果
- 不会删除已生成的中间结果

## 五、完整使用示例

### 场景：训练一个游戏界面检测模型

**目标**：使用标注好的游戏截图数据训练 YOLOv8n 模型

#### Step 1：准备数据集

1. 进入「数据集管理」页面
2. 确保有就绪状态的数据集版本
3. 版本中包含足够的已标注图片（建议 500+ 张）

#### Step 2：创建训练任务

1. 进入「训练管理」页面
2. 点击「**新建训练**」按钮
3. 填写配置：
   - 任务名称：`game_ui_yolov8n_v1`
   - 数据集版本：选择准备好的版本
   - 模型架构：yolov8n
4. 设置参数：
   - Epochs：100
   - Batch Size：16
   - 图片尺寸：640
   - 学习率：0.01
   - Early Stop：15
5. 点击确定

#### Step 3：监控训练进度

1. 点击新任务的「**查看**」按钮
2. 在详情面板观察：
   - 当前 Epoch / 总 Epochs
   - 训练进度条
   - 已用时间
3. 切换不同图表标签页，观察训练曲线

#### Step 4：分析训练效果

1. 训练完成后，查看「mAP」图表
2. 确认 mAP50 达到预期（>70% 为较好）
3. 查看 Loss 曲线是否正常下降
4. 检查日志确认无异常

#### Step 5：导出模型

1. 训练完成后，系统自动保存最佳模型
2. 进入「模型管理」页面
3. 找到训练生成的模型
4. 可查看详情、设为默认或下载备份

## 六、常见问题

### Q1：训练需要多长时间？

**影响因素**：
- 数据集大小
- Epochs 数量
- Batch Size
- GPU 性能

**大致时间**（RTX 3080，数据集 1000 张）：
- 50 Epochs：约 1-2 小时
- 100 Epochs：约 2-4 小时
- 200 Epochs：约 4-8 小时

### Q2：如何判断训练是否正常？

**正常表现**：
- Loss 持续下降
- mAP 持续上升
- 日志无错误信息
- 进度条正常推进

**异常表现**：
- Loss 持续上升或剧烈波动
- mAP 不上升或下降
- 显示 CUDA 内存不足
- 训练突然中断

### Q3：显存不足怎么办？

**解决方法**：
1. 减小 Batch Size（从 16 降到 8 或 4）
2. 减小图片尺寸（从 640 降到 320）
3. 使用更小的模型架构（yolov8n）

### Q4：训练效果不好怎么办？

**排查方向**：
1. 数据集质量：图片是否清晰，标注是否准确
2. 数据量：是否足够（建议 500+ 张/类别）
3. Epochs：是否训练足够（过少欠拟合，过多过拟合）
4. 学习率：是否合适
5. 数据分布：训练集和验证集分布是否合理

### Q5：可以恢复中断的训练吗？

**说明**：目前不支持从中断点恢复，需要重新开始训练。建议使用 Early Stop 机制自动控制训练时长。

### Q6：如何选择合适的 Epochs？

**建议**：
- 简单任务（明显目标）：50-100
- 中等任务（一般目标）：100-200
- 困难任务（小目标、密集场景）：200-500

可以通过观察 mAP 曲线判断：如果曲线还在上升，应继续训练；如果已平稳，可以提前停止。

## 八、训练指标说明

### 8.1 Loss（损失）

| 指标 | 说明 | 理想趋势 |
|------|------|----------|
| Box Loss | 边界框位置回归损失 | 持续下降 |
| Cls Loss | 类别分类损失 | 持续下降 |
| Val Box | 验证集边界框损失 | 持续下降后平稳 |

### 8.2 精度指标

| 指标 | 说明 | 理想值 |
|------|------|--------|
| mAP50 | IoU>50% 时的平均精度 | >70% 较好 |
| mAP50-95 | IoU 从 50% 到 95% 的平均精度 | >50% 较好 |

### 8.3 分类指标

| 指标 | 说明 | 含义 |
|------|------|------|
| Precision | 查准率 | 预测正确的比例 |
| Recall | 查全率 | 目标被检出的比例 |

## 九、数据模型说明

训练管理涉及以下核心数据：

| 字段 | 说明 |
|------|------|

| name | 任务名称 |
| status | 任务状态 |
| base_model_architecture | 基础模型架构 |
| dataset_version_id | 使用的数据集版本 ID |
| epochs | 训练总轮数 |
| current_epoch | 当前轮数 |
| batch_size | 批大小 |
| img_size | 图片尺寸 |
| lr0 | 初始学习率 |
| lrf | 最终学习率 |
| patience | Early Stop 轮数 |
| mosaic | Mosaic 增强参数 |
| mixup | MixUp 增强参数 |
| flip_lr | 水平翻转概率 |
| started_at | 开始时间 |
| completed_at | 完成时间 |

| log_summary | 训练日志摘要（mAP 等） |
| error_message | 错误信息（如果有） |
